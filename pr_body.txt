🎯 **What:** The testing gap addressed
This PR introduces testing for the `getExcerpt` function inside `lib/markdown.ts`, specifically focusing on ensuring that string length bounds checking, null/undefined handling, and string trimming behave correctly when truncating strings to generate excerpts.

📊 **Coverage:** What scenarios are now tested
- Correct handling of `null` and `undefined` arguments, guaranteeing they return empty strings.
- Returning the string unmodified if its length is less than or equal to the specified maximum length.
- Properly truncating strings that exceed the specified length and appending `...`.
- Verifying the `trim()` behavior to make sure trailing whitespaces are removed before appending the ellipses (e.g. `Hello world...` instead of `Hello world ...`).
- Ensuring `stripMarkdown` correctly processes the string before truncation is applied.
- Confirming that the fallback default limit of `150` characters is applied when no limit is provided.

✨ **Result:** The improvement in test coverage
`lib/markdown.ts` function `getExcerpt` is now fully covered by targeted unit tests, ensuring robust length-handling behaviour, avoiding edge case bugs regarding nullish inputs and whitespace formatting. All tests pass consistently using Node.js' native test runner.
