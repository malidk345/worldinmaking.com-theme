## 2024-05-18 - Date Instantiations Inside Render and Sort Loops
**Learning:** React components (like the `PostsView` memoized context) and frontend utilities (like `usePosts`) often loop over items and sort them by date. Calculating `.getTime()` or `unix()` from strings using new `Date()` or `dayjs()` inside the comparison function of `.sort((a,b) => ...)` results in `$O(N \log N)$` redundant date instantiations per item. In large data arrays, this degrades client-side sorting and blocking time.
**Action:** Always pre-calculate dates inside an initial `.map()` traversal ($O(N)$) before applying `.sort()`, utilizing the Schwartzian transform: `.map(item => ({ item, time: dayjs(item.timestamp).unix() })).sort(...).map(m => m.item)`. I applied this optimization successfully in `PostsView`, `usePosts`, and `NotificationCenter`.
## 2025-07-23 - O(N*M) to O(N) in ForumTopicSidebar by memoizing latest posts
**Learning:** Found an O(N*M) performance bottleneck in `ForumTopicSidebar.tsx` where it called `getLastActive` for every channel on every render, running an O(N) `filter` and `reduce` over the `posts` array and repeatedly creating `Date` objects each time. The gallery files are excluded from TS correctly in `tsconfig.json` but still run via `pnpm lint` causing lint errors, and these were pre-existing.
**Action:** Use `useMemo` to construct a memoized `Map` (`Record<number, string>`) in O(N) by traversing the `posts` array just once and doing simple string comparison for `created_at` timestamp. This reduces the complexity from O(N * M) to O(N + M) and avoids creating expensive `Date` objects inside loops.
