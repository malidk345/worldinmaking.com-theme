## 2024-05-18 - Date Instantiations Inside Render and Sort Loops
**Learning:** React components (like the `PostsView` memoized context) and frontend utilities (like `usePosts`) often loop over items and sort them by date. Calculating `.getTime()` or `unix()` from strings using new `Date()` or `dayjs()` inside the comparison function of `.sort((a,b) => ...)` results in `$O(N \log N)$` redundant date instantiations per item. In large data arrays, this degrades client-side sorting and blocking time.
**Action:** Always pre-calculate dates inside an initial `.map()` traversal ($O(N)$) before applying `.sort()`, utilizing the Schwartzian transform: `.map(item => ({ item, time: dayjs(item.timestamp).unix() })).sort(...).map(m => m.item)`. I applied this optimization successfully in `PostsView`, `usePosts`, and `NotificationCenter`.
## 2024-05-18 - Missing Memoization in Custom Hooks
**Learning:** Custom hooks that return derived arrays (like mapping, filtering, or sorting state data) without wrapping the computation in `useMemo` will recreate the array on every render of the component using the hook. This not only causes O(N log N) re-computation on unrelated hook state updates (like changing an active ID), but also breaks referential equality, causing cascading re-renders in memoized child components that depend on the returned array.
**Action:** Always wrap derived, computationally expensive array operations inside custom hooks with `useMemo`, ensuring the dependencies are strictly tied to the raw data they derive from. I successfully applied this optimization to `sortedDocuments` inside `useDocuments`.
