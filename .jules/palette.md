## 2024-06-25 - Automatic aria-labels from Tooltips
**Learning:** In heavily componentized systems, developers frequently use custom Icon-Buttons (like `OSButton`) and pass strings to `tooltip` props, but forget to explicitly pass `aria-label`s. By modifying the base UI component to automatically fallback `aria-label` to the `tooltip` text, we quickly secure a massive baseline of accessibility for icon-buttons across the app. Also, it's critical to ensure custom wrapping elements (like `<Link>`) properly forward `...props` to underlying DOM nodes, otherwise these a11y attributes are silently dropped.
**Action:** Always check custom Button/Link components in design systems to ensure they propagate spread attributes (especially aria-* tags) to the actual DOM elements, and look for opportunities to automate a11y labels from existing descriptive props like `title` or `tooltip`.
